## ðŸ”¹ How MongoDB Works

### 1. Frontend
- Built with technologies like **HTML, CSS, JavaScript**.  
- Modern frameworks: **React.js, Next.js**, Angular, Vue.  
- The frontend sends requests (API calls) to the backend.

---
### 2. Backend (with Drivers)
- Written in languages like **Node.js, PHP, Java, Python, etc.**  
- Uses MongoDB **drivers** to communicate with the MongoDB server.  
- Example: In Node.js, you use `mongoose` to connect and query the database.
---

### 3. MongoDB Server
- The central component that receives queries from the backend via drivers.  
- Processes the queries (read/write/aggregate).  
- Talks to the **Storage Engine**.

---

### 4. Database & Storage Engine
- **Database**: Logical container for collections and documents.  
- **Storage Engine**: Responsible for storing, retrieving, and managing data files.  
  - Default = **WiredTiger** (faster, supports compression, indexing).  
  - Older = **MMAPv1** (deprecated, no longer recommended).  

---

### 5. Data Files
- Actual data is stored as files on disk.  
- Storage Engine handles **read & write operations** on these files.
